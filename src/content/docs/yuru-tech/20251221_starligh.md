## はじめに

気軽に日記やら技術メモを貯めておけるテキストサイトを作りたいと思って調べてみたら、ピッタリな仕組みがあるらしい！<br>
Astro / Starlight？なんかよくわからないけど、やってみるぞぅ。

### 全体の流れ

- npmコマンドを使って、Astro / Starlightをインストール
- そしたらVSCodeを使ってテキストファイルをMarkdown書式で書こう
- ある程度書けたらGitHubにアップロード
- それだけで、ネットの海にテキストサイトが公開できるのだ！

## (1) 環境準備
今回はWindows環境を前提に進めるぜ、必要になるのは以下！<br>
入ってない人は公式サイトからインストールだ。

- **Node.js**：
  - JavaScriptをブラウザの外で動かすための土台、今回はAstro / Starlightを動かすエンジンとして使うのでJavaScript自体は書かないよん
- **npm**：
  - Node.jsをインストールすると使えるようになるコマンド。Node.js上で動くツールの中から、今回使いたい部品（Astro / Starlight）を取得するのだ
- **Git**：
  - ソースコードやテキストファイルを管理するツール
- **GitHub**：
  - アップロードしたファイルをネット上に公開したり、取得できるサービス。今回はアップロードを通してサイトの公開まで行うぞ
- **VSCode**：
  - テキストエディタ、Markdown書式とGitと相性がいい

環境準備というが、この時点ではAstro / Starlight はまだ入っていない。

### Astro / Starlighとはなんぞや？

これから使う「Astro / Starlight」が何者なのかを今のうちに整理しておく。<br>
- **Astro** ：
  - テキストファイル（Markdownなど）を元にして、Webサイトの形（HTML）を作ってくれるツール
- **Starlight** ：
  - Astroで作ったHTMLをいい感じのデザインやら機能性を追加してくれるツール

## (2)作業フォルダを作ろう
VSCodeで早速素敵な日記ファイルを作りたいものだが、ファイルを置く場所がまず必要。<br>
とはいえ難しいものではなく、単純に作業スペースとなるフォルダを作成すればOK👍<br>
ただ、最終的にGithubにアップロードする元ネタフォルダにもなるので意外と大切な工程だ！

```bash
# 作業用フォルダを作成しよう
# 例）Documents 配下に作りたい場合は、まずそこまでcdコマンドで移動する
cd C:\Users\ユーザー名\Documents

# Astro + Starlight のプロジェクトを作成しよう
npm create astro@latest -- --template starlight arigato-neko
cd arigato-neko
```

ちょっと待ってストップ～！！ここは何が何の何？という感じなので…

### npmコマンドで何が起きている？

`npm create astro@latest -- --template starlight arigato-neko`を実行すると、Astro / Starlight を使ったテキストサイトのひな形が、まるごと1セットで作られるぞ。<br>
まず`npm create astro@latest`で、npm に対して「Astroの最新版を使ってプロジェクトを1つ作りたい」意思表示をして、`--template starlight`で「普通のAstroではなくStarlight用に用意された形でプロジェクトを作ってほしい」と指定したという流れ

## (3)GitHubリポジトリ作成 & 初回push
あらかじめGitHubで空リポジトリを作っておこう
```
# ローカルファイルに対してGitの管理対象フォルダ宣言を行う
git init

# 今あるファイル全部を次のコミットに含める宣言を行う
git add .

# 初回コミット
git commit -m "init starlight site"
git branch -M main

# GitHubのURLをremoteに登録
git remote add origin https://github.com/mizunomu1v1/arigato-neko.git

# pushする！
git push -u origin main
```

## (4)astro.config.mjs を GitHub Pages向けに調整
「astro.config.mjs」はAstro プロジェクト全体の設定ファイルだ！
実際に今変えるのはこれぐらい
```
export default defineConfig({
    site: 'https://mizunomu1v1.github.io/arigato-neko/',
    base: '/arigato-neko/',
```
ここを変えた時点で元URLは404になるので、次からはhttp://localhost:4321/arigato-neko/
で開こう

## (5)GitHub Actions（自動デプロイ）を追加
https://docs.astro.build/en/guides/deploy/github/?utm_source=chatgpt.com

## (6)GitHub側の設定（1回だけ）
GitHubのリポジトリ画面でSettings → Pages → Source を「GitHub Actions」に変更しよう
これで、以後は Markdownをcommit/pushしたら自動反映されるというが・・？？
